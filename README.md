# VideoSpeedWheelAndKeyBoard
一款轻量、高效的浏览器扩展，通过**鼠标滚轮**或**键盘快捷键**快速调节主流视频平台的播放倍速，支持多平台个性化配置与速率记忆，提升视频观看效率。

## 项目简介
VideoSpeedWheel 基于 **WXT**（现代化浏览器扩展开发框架） + Vue 3 开发，专为视频爱好者打造，解决不同平台倍速调节操作繁琐的问题。只需将鼠标悬停在倍速显示区域，滚动滚轮即可微调播放速度，也支持键盘快捷键精准设置，适配 B 站、小红书、抖音、快手、YouTube 等主流视频平台。

## 核心特性
🖱️ 滚轮快捷调节：悬停倍速区域，滚轮上下快速调整播放速率
⌨️ 键盘精准控制：Win+Alt+上下键 模拟滚轮调节；Win+Alt+数字（如1.25） 直接设置指定倍速
🔧 个性化配置：每个平台独立配置调节步长、最小 / 最大速率、是否记忆上次速率
📱 多平台适配：支持 B 站、小红书、抖音、快手、YouTube 等主流视频站点
💾 速率记忆：可勾选自动保存每个平台的上次播放速率，下次打开自动生效
🌐 跨浏览器支持：WXT兼容 Chrome、Firefox

## 安装与使用
### 目录结构
```
VideoSpeedWheel/
├── core/                     # 核心逻辑
│   ├── VideoSpeedController.js    # 倍速控制类
│   ├── VideoSpeedConfig.js        # 各平台配置
│   └── VideoSpeedUi.js            # 界面相关
├── entrypoints/
│   ├── content.js             # 注入页面脚本
│   └── popup/                 # 配置弹窗
└── package.json
```
### 环境准备
- Node.js >= 18.x（推荐 20.x LTS 版本）
- npm
- 现代浏览器（Chrome >= 90 / Firefox >= 91）

### 本地开发
1. 克隆项目（或下载源码）
2. 安装依赖
npm install
3. 启动开发环境
npm run dev
npm run dev:firefox
### 构建打包
npm run build
npm run build:firefox
### 打包为zip压缩包（用于发布）
npm run zip          # Chrome
npm run zip:firefox  # Firefox

构建产物会输出到 dist/ 目录：
- Chrome：dist/chrome/
- Firefox：dist/firefox/

### 使用说明
#### 鼠标操作
1. 将鼠标悬停在视频页面的倍速显示区域（如 B 站的「1.0x」位置）
2. 向上滚动滚轮：增加播放倍速（步长可配置，默认 0.1）
3. 向下滚动滚轮：降低播放倍速（步长可配置，默认 0.1）
4. 移开鼠标：自动保存当前倍速（开启记忆功能时）

#### 键盘操作
| 快捷键组合                   | 功能 | 备注 |
|-------------------------|------|------|
| Win+Alt+↑               | 增加播放倍速 | 步长与滚轮一致 |
| Win+Alt+↓               | 降低播放倍速 | 步长与滚轮一致 |
| Win+Alt + 数字（如 1.14514） | 直接设置指定倍速 | 1.5 秒内输入完成生效 |

### 自定义配置
1. 点击浏览器扩展栏中的「VideoSpeedWheel」图标，打开配置弹窗
2. 可配置项：
    - 滚轮调节步长：每次滚动的速率变化量（默认 0.1）
    - 最小 / 最大播放速率：限制倍速调节范围（默认 0.25~16.0）
    - 开启速率记忆：是否保存上次播放速率（默认开启）
    - 默认记忆速率：开启记忆时的初始倍速（默认 1.0）
3. 配置完成后点击「保存配置」，立即生效

## 注意事项
### 环境相关
- Windows 系统下，Chrome 开发模式的用户数据目录需要手动创建：**.wxt/chrome-data**
- 依赖安装失败时，可尝试清除 npm 缓存：npm cache clean --force

### 平台适配
- 各平台的 DOM 结构可能随版本更新变化，若倍速调节失效，需检查 VideoSpeedConfig.js 中的选择器是否需要更新
- 抖音平台因限制，最大倍速被限定为 4.0，修改配置也无法突破

### 权限说明
- 扩展申请 storage 权限：用于保存各平台的个性化配置
- 扩展申请 tabs 权限：用于获取当前标签页 URL，匹配对应平台的配置

## 已知问题
- 部分平台的倍速显示区域可能被广告 / 弹窗遮挡，需关闭遮挡元素后使用
- YouTube 首次加载可能需要等待页面完全渲染后才能创建倍速显示 UI
